## 四、市场状态识别模块（v3.0 最终版）

### 4.1 四状态定义（互斥，必须归入某一状态）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
状态1: TREND_UP（上涨趋势）
条件（必须同时满足）:
  ① BTC价格 > EMA20(1H) > EMA60(1H)
  ② ETH价格 > EMA20(1H) > EMA60(1H)
  ③ RSI(4H) 在 [40, 75] 区间（非超买）
  ④ ATR(14) > ATR_SMA(20) × 0.9（波动率扩张中）

状态2: TREND_DOWN（下跌趋势）
条件（必须同时满足）:
  ① BTC价格 < EMA20(1H) < EMA60(1H)
  ② ETH价格 < EMA20(1H) < EMA60(1H)
  ③ RSI(4H) 在 [25, 60] 区间（非超卖）
  ④ ATR(14) > ATR_SMA(20) × 0.9

状态3: RANGE（区间震荡）
条件（必须同时满足）:
  ① ATR(14) < ATR_SMA(20) × 0.8（波动率收缩至20日均线80%以下）
  ② 布林带宽度（BB_Width）< 过去20日均值 × 0.7（带口收紧）
  ③ BTC和ETH趋势方向不明确（不满足TREND_UP/DOWN任一条件）

状态4: CHAOS（禁止交易）
条件（满足任一即触发）:
  ① BTC和ETH趋势方向矛盾（BTC上涨且ETH下跌，或反之）且持续超过1小时
  ② ATR(14) < ATR_SMA(20) × 0.5（极度低波动）
  ③ 重大新闻窗口：CPI/FOMC/非农公布前2小时至公布后30分钟内
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.2 EMA周期规范（统一标准）

| 用途 | 周期 | 说明 |
|------|------|------|
| 信号EMA | EMA20(1H)、EMA60(1H) | 入场信号判断 |
| 趋势确认 | RSI(4H) | 4H RSI 确认趋势强度 |
| 波动率 | ATR(14) vs ATR_SMA(20) | 判断市场活跃度 |
| 布林带 | BB(20, 2) | 区间波段专用 |

### 4.3 状态切换规则

```
TREND_UP → CHAOS:      停止开多仓，持有可继续
TREND_DOWN → CHAOS:    停止开空仓，持有可继续
RANGE → TREND_UP:      允许做多信号

## 五、信号生成模块（v3.0 最终版）

### 5.1 信号类型与优先级

| 优先级 | 信号类型 | 适用状态 | 持仓规则 |
|--------|----------|----------|----------|
| 1（最高） | 插针反转信号 | RANGE回调段 / 趋势回调段 | 15分钟上限，超时强制平 |
| 2 | 趋势顺势信号 | TREND_UP / TREND_DOWN | 无时间限制，持有至止盈/止损 |

### 5.2 趋势顺势信号

#### 做多（TREND_UP状态）

```
入场条件（必须同时满足）:
  ① ETH价格 > EMA20(1H) > EMA60(1H)      （三重过滤：价格在EMA之上且EMA多头）
  ② RSI(1H) ≤ 60，且 RSI(4H) ≤ 70        （未超买，有上涨空间）
  ③ RSI(1H) 刚从超卖区(<35)回升，或在 [40, 50] 区间
  ④ 成交量 > 过去20根(1H)均量 × 1.2      （量能确认）
  ⑤ 回调不破前低（价格回踩EMA20不破最佳）

入场执行:
  首仓 50% → 市价立即入场
  次仓 50% → 限价挂单 = 市价 × (1 - 0.15%)
  次仓有效期: 5分钟，未成交撤单

杠杆: 30x
```

#### 做空（TREND_DOWN状态）

```
入场条件（必须同时满足）:
  ① ETH价格 < EMA20(1H) < EMA60(1H)      （三重过滤）
  ② RSI(1H) ≥ 40，且 RSI(4H) ≥ 30        （未超卖，有下跌空间）
  ③ RSI(1H) 刚从超买区(>65)回落，或在 [50, 60] 区间
  ④ 成交量 > 过去20根(1H)均量 × 1.2
  ⑤ 反弹不过EMA20（价格反弹至EMA20滞涨最佳）

入场执行:
  首仓 50% → 市价立即入场
  次仓 50% → 限价挂单 = 市价 × (1 + 0.15%)
  次仓有效期: 5分钟，未成交撤单

杠杆: 30x
```

  A. RSI(5分钟) < 30 或 RSI(15分钟) < 40
  B. 下影线时段成交量明显放大
  C. 价格触及布林下轨 或 日内重要支撑位
  D. 形态：锤子 / 蜻蜓 / 孕育线

入场执行:
  首仓 50% → 市价立即入场
  次仓 50% → 限价挂单 = 入场价 × (1 - 0.15%)
  次仓有效期: 5分钟，未成交撤单

## 六、交易执行模块（v3.0 最终版）

### 6.1 仓位计算

```
账户配置:
  总资金 = 实时净值 USDT
  单笔仓位上限 = 总资金 × 2%
  最大总仓位 = 总资金 × 10%（5笔单）
  合约面值: ETH-USDT-SWAP = 0.01 ETH/张

下单数量计算:
  做多/做空张数 = (总资金 × 2%) ÷ (入场价 × 合约面值 ÷ 杠杆)

默认杠杆:
  趋势顺势信号: 30x
  插针反转信号: 50x
```

### 6.2 分批入场规则

```
入场批次:
  第1批: 50%仓位，市价立即入场
  第2批: 50%仓位，限价挂单优化成本

限价单参数:
  做多挂单价 = 市价 × (1 - 0.15%)
  做空挂单价 = 市价 × (1 + 0.15%)
  有效期: 5分钟
  超时未成交: 撤单，不追

总仓位控制:
  同一方向最多2笔（总4%仓位）
  同时持有多空方向最多各1笔（总4%仓位）
  任意时刻总仓位不超过总资金10%
```

### 6.3 加权平均入场价计算（多批入场）

```
当分批入场时，止损/止盈/移动止损全部基于加权平均价：

加权平均价 = (第1批数量 × 第1批价格 + 第2批数量 × 第2批价格) ÷ 总数量

例:
  首仓 50张 @ 1000 U
  次仓 50张 @ 998 U（挂单成交）
  → 加权平均价 = (50×1000 + 50×998) ÷ 100 = 999 U
  → 止损 = 999 × 0.985 = 984.015 U
```

## 七、止损规则（v3.0 最终版）

### 7.1 固定止损

```
止损幅度: 1.5%（固定，不可调整）

做多止损 = 入场均价 × (1 - 0.015)
做空止损 = 入场均价 × (1 + 0.015)

执行: 触及止损价，市价全出，不等待
```

### 7.2 移动止损（保本规则）

```
盈利 ≥ +1.0% 时:
  做多: 止损移动到 入场均价（保本）
  做空: 止损移动到 入场均价（保本）

盈利 ≥ +1.5% 时:
  做多: 止损移动到 入场均价 × (1 + 0.5%)
  做空: 止损移动到 入场均价 × (1 - 0.5%)
```

### 7.3 止损执行原则

```
① 触及止损价 → 市价全出，不等待、不犹豫
② 止损后不补仓，不扛单
③ 止损记录立即存档，用于统计连续亏损
```

## 八、止盈规则（v3.0 最终数学版）

> ⚠️ 核心原则：每次止盈以"当前剩余持仓"为分母计算平仓数量

### 8.1 标准止盈（3批）

```
假设: 开仓 N 张，入场均价 P

【第一止盈：盈利 +0.5%】
  当前持仓: N 张
  平仓数量: N × 30%
  剩余持仓: N × 70%
  止损移动到: P（保本）

【第二止盈：盈利 +1.0%】
  当前持仓: N × 70% 张（上次止盈后的剩余）
  平仓数量: (N × 70%) × 40% = N × 28%
  累计已平: N × 58%
  剩余持仓: N × 42%
  止损移动到: P × 1.005（+0.5%）

【第三止盈：盈利 +1.5%】
  当前持仓: N × 42% 张
  平仓数量: 剩余全部 = N × 42%
  累计已平: N × 100%（全平）
  操作结束
```

### 8.2 趋势延续特别处理（可选）

```
触发条件: 信号评分 ≥ 8分（强趋势信号）

第一止盈 +0.5% 时:
  → 只平 20%（而非标准30%）
  → 剩余 80% 持仓
  → 止损移动到 P（保本）

第二止盈 +1.0% 时:
  → 平 50%
  → 累计已平 70%
  → 剩余 30%
  → 止损移动到 P × 1.005

第三止盈 +1.5% 时:
  → 平剩余 30%
  → 全平，结束

趋势延续判断:
  若价格继续单边移动，止损持有不出
  直到趋势反转信号出现 → 触发第三止盈
```

### 8.3 回调止盈（任选其一）

```
A. 盈利 +1.5% 后出现 0.5% 回调 → 全平
B. 盈利 +2.0% 后出现 0.75% 回调 → 全平
```

### 10.2 风控铁律（永久执行）

```

② 单日亏损 ≥ 5% → 当日停止交易，不允许开新仓

⑧ 保本优先 → 盈利1%即保本，不贪婪

### 10.3 连续亏损处置

```
连续亏损定义: 连续3次止损/超时平仓

处置流程:
  STEP 1: 触发第3次亏损 → 立即停止开仓
  STEP 2: 进入冷却期 1小时
  STEP 3: 冷却结束后，先分析市场状态
  STEP 4: 若状态为 CHAOS → 继续等待，不开仓
  STEP 5: 若状态非 CHAOS → 等待信号评分 ≥ 8分 才可开仓

冷却期禁止:
  ✗ 不开新仓
  ✗ 不补仓
  ✗ 不手动干预现有持仓
```

    # STEP 1: 获取数据
    # ====================
    btc_data   = get_btc_ohlcv()      # EMA20/60(1H), RSI(1H/4H), ATR(14)
    eth_data   = get_eth_ohlcv()      # 价格, EMA20/60(1H), RSI(1H/4H), ATR, 成交量
    account    = get_account()        # 余额, 持仓, 可用保证金

    # ====================
    # STEP 2: 判定市场状态
    # ====================
    regime = detect_market_regime(btc_data, eth_data)
    # 输出: TREND_UP / TREND_DOWN / RANGE / CHAOS

    # ====================
    # STEP 3: CHAOS 过滤
    # ====================
    if regime == CHAOS:
        # 不开新仓，现有持仓继续监控
        monitor_positions()
        sleep(30)
        continue

    # ====================
    # STEP 4: 信号生成
    # ====================
    signal, score = generate_signal(regime, eth_data)
    # 输出: LONG/SHORT/NONE,  score: 0-10

    if signal == NONE:
        sleep(30)
        continue

    # ====================
    # STEP 5: BTC 方向过滤
    # ====================
    btc_direction = get_btc_direction(btc_data)
    if btc_direction != signal:
        sleep(30)
        continue

    # ====================
    # STEP 6: 交易过滤器
    # ====================
    if not all_filters_pass(btc_data, eth_data, account, regime, score):
        sleep(30)
        continue

    # ====================
    # STEP 7: 确认仓位
    # ====================
    if score >= 8:
        position_size = TOTAL_CAPITAL * 0.02    # 满仓 2%
    elif score >= 5:
        position_size = TOTAL_CAPITAL * 0.01    # 半仓 1%
    else:
        sleep(30)
        continue

    # ====================
    # STEP 8: 用户确认（必须步骤）
    # ====================
    send_confirmation_to_user(signal, position_size, score)
    # ⚠️ 等待用户回复"确认"后才执行，不自动下单

    # ====================
    # STEP 9: 执行交易
    # ====================
    if user_confirmed:
        execute_order(signal, position_size)
        set_stop_loss()
        set_take_profit()
        log_order()

    # ====================
    # STEP 10: 持仓监控（每30秒）
    # ====================
    while has_open_positions():
        check_stop_loss()      # 触及止损 → 市价全出
        check_take_profit()    # 触及止盈 → 分批平仓
        check_timeout()        # 超时 → 强制平仓
        check_trend_reverse()  # 趋势反转 → 平仓
        sleep(30)

    # ====================
    # STEP 11: 每日风控检查
    # ====================
    if is_new_day():
        if daily_loss >= TOTAL_CAPITAL * 0.05:
            disable_trading()  # 当日停止交易
        check_consecutive_losses()

    sleep(30)
END WHILE
