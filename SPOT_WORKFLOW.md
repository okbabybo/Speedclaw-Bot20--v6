## 一、交易前准备（每日开盘前必做）

### 1.1 账户健康检查（每次启动都要做）

```
□ API连接测试
  └─ get_balance("USDT") → 返回正常余额
  └─ get_klines("BTCUSDT", "1h", 5) → 返回5根K线
  └─ get_orderbook("BTCUSDT") → 返回买卖盘口

□ 余额确认
  └─ 余额 < $11 → 报警，禁止开仓
  └─ 余额 $11-20 → 仅开1个币，限额$20
  └─ 余额 $20-50 → 每币$20，最多1个
  └─ 余额 $50-200 → 每币$50
  └─ 余额 $200-1000 → 每币$150
  └─ 余额 > $1000 → 每币$300

□ 状态恢复检查
  └─ 读取 grid_state.json
  └─ 对比API真实持仓 vs 状态文件
  └─ 发现差异 → 识别手动平仓 → 记录日志 → 释放资金
```

### 1.2 市场环境判断（开盘前5分钟）

```
□ BTC 走势定方向
  └─ BTC EMA20(4H) 向上 → 大盘多头
  └─ BTC EMA20(4H) 向下 → 大盘空头
  └─ BTC EMA20(4H) 走平 → 震荡

□ 波动率检查
  └─ BTC 日内波幅 > 8% → 高波动模式
  └─ BTC 日内波幅 3-8% → 正常模式
  └─ BTC 日内波幅 < 3% → 低波幅，减少开仓

□ 资金费率检查（币安合约）
  └─ BTC 资金费率 > 0.05% → 市场过热，减少做多
  └─ BTC 资金费率 < -0.05% → 市场看空，减少做空
```

## 二、市场扫描工作流（每3分钟一次）

### 2.1 数据采集（每次扫描）

```
对每个币种 BTC, ETH, BNB, SOL, AVAX, XRP 同时采集：
  ├─ 1H K线 × 200根  → 计算 RSI(1H), EMA20, MACD, BB
  ├─ 4H K线 × 100根  → 计算 RSI(4H), EMA20
  ├─ 1D K线 × 30根   → 计算 RSI(1D)
  └─ 1H K线 × 50根   → 计算 ATR
```

### 2.2 指标计算

```
对每个币种并行计算：
  ├─ RSI(14) = RSI(c1h, 14)
  ├─ EMA(20) = calc_ema(c1h, 20)
  ├─ EMA_H4 = calc_ema(c4h, 20)
  ├─ EMA_D1 = calc_ema(c1d, 20)
  ├─ MACD(12,26,9) → macd_line, signal_line, histogram
  ├─ BB(20,2) → upper, mid, lower
  ├─ BB_position = (price - lower) / (upper - lower)
  ├─ ATR = calc_atr(klines_1h, 14)
  └─ Vol_ratio = volume_1h / avg(volume_1h[-20:])
```

### 2.3 模式判断流程

```
                    获取 RSI_1H, RSI_4H, RSI_1D
                              │
              ┌───────────────┼───────────────┐
              ↓               ↓               ↓
        RSI_1D > 80?    RSI_1D < 20?    其他情况
        或 RSI_1H > 85       或 RSI_1H < 15
              │               │               │
              ↓               ↓               ↓
         CRISIS          CRISIS          检查EMA方向
         全部暂停        全部暂停           │
                                   ┌────────┼────────┐
                                   ↓        ↓        ↓
                             EMA三周期   EMA走平   其他
                             全部向上             ↓
                                   │      RANGE_BOUND
                                   ↓
                              ┌─────┴─────┐
                              ↓           ↓
                          向上↑        向下↓
                           │           │
                           ↓           ↓
                       TREND_UP   TREND_DOWN

    高波动条件：日内波幅 > 8% + RSI < 35 + 放量 → VOLATILE_OVERSOLD
    高波动条件：日内波幅 > 8% + RSI > 65 + 放量 → VOLATILE_OVERBOUGHT
```

## 三、信号判定工作流

### 3.1 网格买入信号（Grid BUY）

**触发条件（同时满足）：**
```
① RSI_1H < 40
② BB_position < 0.3（靠近下轨）
③ 模式 ≠ TREND_DOWN（不在下跌趋势中）
④ 没有已持有的网格仓位
⑤ 余额 ≥ $11
⑥ 活跃仓位 < 3个币
```

**加分项（满足越多越好）：**
```
+ RSI_1H < 35 → 深度超卖，优先度高
+ BB_position < 0.2 → 触及下轨
+ Vol_ratio > 1.3 → 放量确认
+ RSI_4H < 50 → 4H也有超卖
+ MACD histogram > 0 → 多头背离
```

**下单金额计算：**
```
基础金额 = min(余额 / 活跃币种数, 余额 × 30%)
熊市打折 = × 0.5（TREND_DOWN模式）
实际金额 = max(基础金额, $11)

下单数量 = 实际金额 / 当前价格
数量精度 = floor(数量 / LOT_SIZE) × LOT_SIZE
```

**下单执行：**
```
1. 执行: market_buy(symbol, quantity)
2. 记录: {
     buy_price: 成交价,
     qty: 成交数量,
     target: buy_price × 1.006,  // +0.6% 止盈
     sl: buy_price × 0.88,       // -12% 止损
     ts_triggered: false,
     ts_price: 0,
     mode: 当前模式,
     bought_at: timestamp
   }
3. 日志: [网格买入] BTCUSDT格2 @ $62000 qty=0.00150
4. 启动止盈止损监控
```

### 3.2 趋势买入信号（Trend BUY）

**触发条件（同时满足）：**
```
① 模式 = TREND_UP 或 VOLATILE_OVERSOLD
② 趋势评分 Trend_score ≥ 5
③ EMA20_1H 向上
④ 没有已持有的趋势仓位
⑤ 余额 ≥ $30（趋势需要更多资金）
⑥ 活跃仓位 < 3个币
```

**加分项：**
```
+ RSI_1H 在 40-60（健康多头区间）+2分
+ MACD histogram > 0 +2分
+ Vol_ratio > 1.3 +1分
+ BB_position 在 0.4-0.7（BB中段）+1分
+ RSI_4H > 55 +1分
```

**下单金额计算：**
```
实际金额 = min(余额 × 30%, 每币上限)
下单数量 = 实际金额 / 当前价格
```

**下单执行：**
```
1. 执行: market_buy(symbol, quantity)
2. 记录: {
     buy_price: 成交价,
     qty: 成交数量,
     entry: 成交价,
     peak: 成交价,           // 跟踪最高价
     sl: buy_price × 0.88,  // -12% 止损
     tp1: buy_price × 1.15, // +15% 第一止盈（50%仓位）
     tp2: buy_price × 1.25, // +25% 第二止盈（剩余50%）
     tp1_done: false,
     opened_at: timestamp
   }
3. 日志: [趋势买入] ETHUSDT @ $1660 qty=0.0602
4. 启动趋势止盈止损监控
```

### 3.3 平仓信号（SELL）

**触发条件（满足任一即平仓）：**

```
网格仓位：
  ① 价格 ≥ 目标止盈价 → 止盈卖出
  ② 价格 ≤ 止损价 → 止损卖出
  ③ 利润 > 6% 且 追踪止损触发 → 追踪卖出
  ④ 模式 = VOLATILE_OVERBOUGHT → 全部止盈
  ⑤ 模式 = CRISIS → 全部止损
  ⑥ 模式 = TREND_DOWN → 全部止损

趋势仓位：
  ① 价格 ≥ TP2 → 全仓止盈
  ② 价格 ≥ TP1 且 tp1_done = false → 50%止盈
  ③ 价格 ≤ SL → 止损卖出
  ④ 峰值利润 > 20% 且 回撤 ≥ 8% → 追踪止损
  ⑤ 峰值利润 > 15% 且 回撤 ≥ 5% → 追踪止损
  ⑥ 价格 < EMA20 × 0.96 → 趋势破坏止损
  ⑦ RSI_1H > 75 → 过热止盈
```

## 四、止盈止损执行工作流

### 4.1 网格止盈止损（每20秒检查一次）

```
┌─────────────────────────────────────────────┐
│  价格变化触发检查                           │
└──────────────┬──────────────────────────────┘
               ↓
    ┌──────────┼──────────┐
    ↓          ↓          ↓
  价格≥目标  价格≤止损  其他
    │          │          │
    ↓          ↓          ↓
  网格止盈  网格止损  检查追踪
               │          │
               ↓          ↓
           记录亏损   利润>6%？
           连亏计数      │
           更新新高      ↓
                      追踪激活
                      回撤>3%？
                         │
                         ↓
                    追踪卖出
```

**网格止盈执行：**
```
1. 执行: market_sell(symbol, qty)
2. 记录: {
     sold: true,
     sold_price: 成交价,
     sold_reason: "TP+0.6%",
     sold_at: timestamp
   }
3. 计算利润: (sold_price - buy_price) / buy_price × 100%
4. 日志: [网格卖出] BTCUSDT格3 @ $62400 (+1.2%) TP+0.6%
5. 资金释放: buy_price × qty 加回可用余额
6. 盈利记录: record_win()（清零连亏计数）
```

**网格止损执行：**
```
1. 执行: market_sell(symbol, qty)
2. 记录: {
     sold: true,
     sold_price: 成交价,
     sold_reason: "SL",
     sold_at: timestamp
   }
3. 计算亏损: (sold_price - buy_price) / buy_price × 100%
4. 日志: [网格止损] BTCUSDT格3 @ $55000 (-8.5%) SL
5. 资金释放: buy_price × qty 加回可用余额
6. 连亏记录: record_loss()（+1连亏计数）
```

### 4.2 趋势止盈止损（每20秒检查一次）

```
┌─────────────────────────────────────────────┐
│  价格变化触发检查                           │
└──────────────┬──────────────────────────────┘
               ↓
    ┌──────────┼──────────┬──────────┐
    ↓          ↓          ↓          ↓
  价格≥TP2  价格≥TP1  价格≤止损  其他
    │          │          │          │
    ↓          ↓          ↓          ↓
  全仓止盈  50%止盈  止损卖出   更新峰值
                   │          │
                   ↓          ↓
              tp1_done=true  更新峰值
                              计算回撤%
                                 │
                    ┌────────────┼────────────┐
                    ↓            ↓            ↓
               回撤≥8%?    回撤≥5%?     回撤≥3%?
               (利润>20%) (利润>15%) (利润>10%)
                    │            │            │
                    ↓            ↓            ↓
               追踪卖出    追踪卖出    追踪卖出
```

**趋势止盈执行（分批）：**
```
TP1触发（+15%，50%仓位）：
1. 卖出数量 = qty × 50%
2. 执行: market_sell(symbol, partial_qty)
3. 记录: tp1_done = true
4. 日志: [趋势分批止盈] ETHUSDT @ $1909 (+15%) 50%仓位

TP2触发（+25%，剩余50%仓位）：
1. 卖出数量 = 剩余全部
2. 执行: market_sell(symbol, remaining_qty)
3. 仓位清空: position = null
4. 日志: [趋势止盈] ETHUSDT @ $2075 (+25%) 全部仓位
5. record_win()
```

**趋势止损执行：**
```
1. 执行: market_sell(symbol, qty)
2. 计算: (sold_price - entry) / entry × 100%
3. 日志: [趋势止损] ETHUSDT @ $1440 (-12%) SL触发
4. 仓位清空: position = null
5. record_loss()
```

### 4.3 追踪止损执行

```
利润计算: current_profit = (current_price - entry_price) / entry_price

追踪激活条件:
  网格: current_profit > 6% → 激活追踪止损
  趋势: current_profit > 15% → 激活追踪止损

追踪触发条件:
  网格: current_price < peak_price × (1 - 3%)
  趋势(利润>20%): current_price < peak_price × (1 - 8%)
  趋势(利润>15%): current_price < peak_price × (1 - 5%)
  趋势(利润>10%): current_price < peak_price × (1 - 3%)
```

## 五、资金管理全流程

### 5.1 开仓资金计算

```
余额 = get_balance("USDT")
活跃币种数 = 已开仓网格数 + 已开仓趋势数

每币最大额度:
  if 余额 < 50:    limit = 20
  elif 余额 < 200:  limit = 50
  elif 余额 < 1000: limit = 150
  else:            limit = 300

实际每币额度 = min(余额 / 活跃币种数, limit, 余额 × 30%)
下单金额 = 实际每币额度 × 模式系数
```

**模式系数：**
```
TREND_UP:           × 1.0  （全力做多）
RANGE_BOUND:        × 0.8  （稳健网格）
VOLATILE_OVERSOLD:  × 1.2  （透支20%）
TREND_DOWN:         × 0.3  （轻仓试探）
VOLATILE_OVERBOUGHT: × 0   （禁止开仓）
CRISIS:             × 0    （禁止开仓）
```

### 5.2 复利滚动流程

```
┌─────────────────────────────────────────────┐
│           每笔利润分配决策                   │
└──────────────┬──────────────────────────────┘
               ↓
    ┌──────────┼──────────┐
    ↓          ↓          ↓
 网格止盈   趋势止盈   止损亏损
    │          │          │
    ↓          ↓          ↓
 100%再投资  100%再投资  不补资金
    │          │          │
    ↓          ↓          ↓
 自动分配   自动分配   连亏计数
 到下一格    到下一仓     │
               │          ↓
               ↓      达到3次
            利润滚存    暂停15分
```

### 5.3 提盈流程

```
触发条件: 余额 ≥ 历史最高 × 1.20
最低提盈: 利润 ≥ $5
提盈比例: 利润的 50%

执行:
1. 计算: 可提盈 = (余额 - 高点) × 50%
2. 日志: 💰 提盈! +$15.50 | 余额$115.50 | 高点$100.00
3. 更新高点: 高点 = 余额 × 0.90（提盈后保留90%）
4. 累计提盈 += 可提盈
5. 资金状态: 已提取，不参与复利
```

## 六、风控触发工作流

### 6.1 熔断机制（连亏3次）

```
每次止损触发:
  连亏计数 += 1
  连亏时间 = now

熔断检查:
  if 连亏计数 >= 3:
    if now - 连亏时间 < 15分钟:
      → 禁止开仓
      → 日志: [熔断] 连亏3次，暂停X分钟
    else:
      → 重置计数 = 0（超过时间窗口，清零）
```

### 6.2 回撤保护（-20%）

```
每次余额更新时:
  if 余额 < 历史最高 × 0.80:
    → 执行全部止损（网格 + 趋势）
    → 锁定30分钟禁止开仓
    → 日志: [⚠️ 回撤保护] $100→$78，清仓锁定
    → 重置高点 = 当前余额 × 0.90
```

### 6.3 单日最大亏损（-8%）

```
每日凌晨重置: 当天最大亏损 = 0

余额亏损检查:
  日内亏损% = (高点 - 当前) / 高点
  if 日内亏损% > 8%:
    → 暂停1小时
    → 日志: [⚠️ 单日止损] 日内亏9.2%，暂停1小时
    → 锁定1小时后自动恢复
```

### 6.4 手动平仓识别

```
每5分钟一次检测:
  状态文件持仓 = grid_engines[sym].positions
  API真实持仓 = get_my_trades(sym, limit=10)

  for each grid_index in 状态文件:
    if 状态=持有 but API无对应成交:
      → 手动平仓识别
      → 日志: [⚠️ 手动平仓识别] BTCUSDT格2 @ $62000
      → 标记 sold=True, manual_close=True
      → 释放资金到可用余额
      → 不触发追踪止损
```

## 九、盈利精准操作清单

```
□ 每次开仓前：确认余额 ≥ $11
□ 每次开仓前：确认模式允许开仓
□ 每次开仓：记录 entry / SL / TP
□ 每20秒：检查所有仓位止盈止损
□ 每次止盈：50%仓位 → TP1，50%仓位 → TP2
□ 每次止损：record_loss() → 连亏+1
□ 每次止盈：record_win() → 连亏清零
□ 余额创新高：更新 high_water
□ 余额达到高点×120%：提盈50%
```
