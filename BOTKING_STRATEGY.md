## 三、市场模式（7种，新增趋势回调）

BotKing 根据技术指标自动判断当前市场属于哪种模式，然后执行对应策略。

### 判断逻辑

```
日线EMA20 + 4H EMA20 + 1H EMA20 + RSI + 波幅 + 成交量
```

### 模式判断规则

| 模式 | 日线 | 4H | 1H | RSI | ADX | 信号 |
|------|------|----|----|-----|-----|------|
| **TREND_UP** | EMA上 | EMA上 | EMA上 | 任意 | >25 | 🟢 买 |
| **TREND_UP_RECALL** | EMA上 | EMA向上 | RSI<40 | >20 | 📈 回调买 |
| **TREND_DOWN** | EMA下 | EMA下 | <50 | >25 | 📉 轻仓/观望 |
| **RANGE_BOUND** | 震荡 | 震荡 | 40-60 | <20 | 📊 网格 |
| **VOL_OVERSOLD** | 放量大跌 | - | <35 | >30% | 爆发 | 🔴 网格+趋势 |
| **VOL_OVERBOUGHT** | 放量大涨 | - | >65 | >30% | 爆发 | 🟠 全部止盈 |
| **CRISIS** | RSI>80 | - | RSI<20 | 极端 | 任意 | 💥 全暂停 |

### 模式识别指标详解

```
EMA20(指数移动平均线):
  - 计算最近20根K线收盘价的加权平均
  - 价格 > EMA20 → 短期偏多
  - 价格 < EMA20 → 短期偏空

ADX(趋势强度指数，0-100):
  - ADX > 25：强趋势，适合趋势追踪
  - ADX < 20：震荡，适合网格套利
  - ADX 20-25：中性，判断其他指标

RSI(相对强弱指数):
  - RSI > 70：超买，价格可能回调
  - RSI < 30：超卖，价格可能反弹
  - RSI = 50：中性

RSI底背离：
  - 价格创新低但RSI没有创新低
  - = 潜在反弹信号，BotKing自动加分

波幅(ATR):
  - ATR > 5%：高波动，适合少格宽距
  - ATR 2-5%：中波动，标准网格
  - ATR < 2%：低波动，适合多格窄距

成交量:
  - 当前成交量 > 30日均量×1.3 → 放量信号有效
  - 当前成交量 < 均量70% → 信号可能不可靠

Fear & Greed指数(0-100):
  - < 25：极度恐慌 → 超卖信号置信度+15%
  - > 75：极度贪婪 → 超买信号置信度+10%
  - 数据来源：alternative.me，每小时更新
```

## 四、双引擎说明

### 引擎1：网格引擎（GridEngine）

**什么时候用？**
- RANGE_BOUND（震荡市）
- VOLATILE_OVERSOLD（超卖反弹）

**原理：**
把资金分成N格，每格等距分布。价格跌到某格就买，涨回来就卖，反复收割。

```
例子（6格，间距0.6%）:

格5  [=====止盈====>]  +0.6%  卖
格4  [===========>    ]  +0.4%  卖
格3  [中心===========]  成本价
格2  [<===========    ]  -0.4%  买
格1  [<=============  ]  -0.6%  买
格0  [<===============]  -1.0%  买

价格跌破格0 → 触发止损（-12%）
```

**网格参数（ATR自适应）：**

| 市场波动 | 格数 | 每格利润 | 止损 |
|---------|------|---------|------|
| 高ATR>5% | 2格 | 1.0% | -12% |
| 中ATR 2-5% | 4格 | 0.6% | -12% |
| 低ATR<2% | 6格 | 0.4% | -12% |

**分批建仓：**
- 第一批：先开1-2格（根据资金量）
- 第二批：止盈确认后，等5分钟再开下一批
- 效果：避免一次性暴露全部资金

**追踪止损（TS）：**
- 浮盈 > 6% 时激活
- 追踪止损线跟随价格上移（创新高就上调）
- 价格从最高点回落 3% → 触发TS，全格卖出
- 效果：锁定利润，不让煮熟的鸭子飞走

---

### 引擎2：趋势引擎（TrendEngine）

**什么时候用？**
- TREND_UP（上升趋势）
- VOLATILE_OVERSOLD（超卖反弹后可能转趋势）

**原理：**
EMA确认方向，持有直到趋势破坏。不做短线高卖低买，吃完整段趋势。

```
买入信号：
  - EMA20(1D+4H+1H) 三周期全部向上
  - RSI 1H > 40

持有：
  - 跟踪价格走势
  - 不做任何止盈/止损操作

卖出信号：
  - 价格 < EMA20×0.96（趋势破坏）
  - 或 RSI > 75（过热）
```

**止盈分两阶段：**

| 阶段 | 条件 | 操作 | 剩余仓位 |
|------|------|------|---------|
| TP1 | 浮盈 ≥ +15% | 卖 50% 仓位 | 50% |
| TP2 | 浮盈 ≥ +25% | 卖 100% 仓位 | 0% |

**趋势追踪止损（TS）：**
- 浮盈 > 15% 后激活
- 追踪止损线跟随价格上移（创新高就上调）
- 激活后价格回落 5% → 触发TS，全仓卖出

**止损：**
- 固定 -12% 止损
- 趋势破坏立即止损

## 七、止盈与复利

### 利润分层（每笔止盈）

```
利润 = 100%
├── 50% 锁定 → 保留不动，不投入市场
├── 30% 复利 → 自动投入下一格/下一仓
└── 20% buffer → 覆盖手续费、滑点
```

### 提盈机制（达到目标时）

```
触发条件：余额 ≥ 历史最高 × 120%
最低门槛：利润 ≥ $5
提取比例：利润的 50%
新历史高点：提取后 × 90%

例子：
  初始 $1000 → 涨到 $1240
  提盈：(1240-1000)×50% = $120
  实际到账：$120
  剩余继续滚：$1120
```

## 八、手动平仓识别

**原理：** 每5分钟对比"状态文件记录的持仓" vs "币安API实际持仓"

**检测逻辑：**
```
状态文件记录 BTC 持仓量: 0.01
API查询实际 BTC 持仓量: 0.005
→ 用户手动卖出了一半 → 识别为手动平仓
→ 标记状态文件，释放对应资金
→ 冷却期5分钟防止重复识别
```

## 九、完整交易流程

### 开盘流程（BotKing自动执行）

```
① 扫描6个币种，检测市场模式
② 按 score 排序买入信号
③ 分配资金开仓
④ 每20秒检查止盈/止损/TS
⑤ 每180秒重新扫描市场模式
⑥ 仓位平仓后自动清理引擎，可重新开仓
```

### 平仓信号

```
自动止盈（网格）：
  - 价格触及目标价 → 卖出该格

自动止损（网格）：
  - 价格触及SL价 → 卖出该格

追踪止损（网格）：
  - 浮盈>6%激活 → 从高点回落3% → 卖出

市场信号平仓：
  - VOL_OVERBOUGHT → 全部止盈
  - TREND_DOWN → 网格全平
  - CRISIS → 全部暂停

手动平仓：
  - 用户在币安APP手动卖出 → BotKing识别并释放资金
```
